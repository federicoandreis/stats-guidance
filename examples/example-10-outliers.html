28  Example 10: Care Home Staffing Investigation

Outlier Detection and Treatment with Data Quality Issues

28.1 Scenario Overview

Method: Outlier detection and treatment Complexity: Medium Time to complete: 2-3 hours QASQA questions: Q3 (Analysis); Q5 (Fitness for Purpose)

NoteMethod Reference

For the detection toolkit itself - the three methods, their personalities, and the agreement triage - see outlier detection methods; for the investigate-first discipline and treatment decisions, see unusual observations.

NoteRunning This Example

These examples require R or Python. If you are not yet set up with a coding environment:

  • Microsoft Fabric: CQC’s primary analytics platform – contact your function’s data engineer for access
  • RSS training: Introductory R and Python courses available via CQC’s learning platform
  • Quant Clinic: Drop-in sessions for hands-on help with code and methods – see the Guild Teams channel for upcoming dates

Both R and Python versions are provided throughout. Use whichever you are more comfortable with.

28.2 Scenario

You are a CQC analyst investigating staffing levels at adult social care homes. Recent submissions show some extreme staff-to-resident ratios that warrant investigation. The analysis will support:

  • Identifying homes requiring immediate inspection (data errors vs genuine understaffing)
  • Understanding natural variation in staffing models
  • Informing risk-based targeting

Analytical Question: Which care homes have staffing ratios extreme enough to warrant investigation, and which extreme values are data errors versus genuine operational issues?

Context: Care homes report monthly staffing data. Most homes have 2-4 qualified staff per 10 residents, but you’ve seen values ranging from 0 to 12. Some may be data entry errors (wrong decimal point, missing zeros), others may reflect genuine operational models (specialist dementia units) or problems (understaffing due to recruitment issues).

28.3 Learning Objectives

By working through this example, you will learn to:

  1. Visualise data to understand distribution before detection
  2. Apply multiple detection methods (Z-score, Modified Z-score, IQR)
  3. Compare method results to prioritise investigation
  4. Investigate flagged observations systematically
  5. Distinguish data errors from genuine extremes
  6. Document outlier treatment decisions for QA purposes
  7. Communicate findings to inspection teams
  8. Handle small provider variation appropriately

28.4 Dataset Description

Synthetic Data: Generated for this example

337 care homes with the following variables:

Variable Description Type Notes
home_id Unique home identifier String Format: CH001-CH350
home_name Care home name String Synthetic names
region NHS England region String 7 regions
beds Number of beds Integer 15-120 beds
care_type Specialisation String General, Dementia, Nursing
cqc_rating Current CQC rating String Outstanding to Inadequate
staff_per_10 Qualified staff per 10 residents Numeric Ratio (target 2.5-4.0)

Data Characteristics

The planted truth (knowing the right answer is what makes a simulation instructive): - Most homes (300): typical staffing around 3.2 staff per 10 residents (standard deviation 0.8) - Specialist units (15): genuinely higher staffing around 5.5 - Struggling homes (10): genuinely low staffing around 1.5 - Planted problem values (12): impossible entries near zero, suspected wrong-unit entries above 9, and a few high-but-plausible returns - Small homes get extra noise: higher natural variation

28.7 Step 3: Apply Detection Methods

  • R
  • Python
# Z-score
care_homes$z_score <- scale(care_homes$staff_per_10)[,1]
care_homes$outlier_z <- abs(care_homes$z_score) > 2.5

# Modified Z-score. NOTE: R's mad() is already scaled to mimic the standard
# deviation (constant = 1.4826), so use the raw MAD (constant = 1) with the
# 0.6745 multiplier - applying both scalings under-flags by a third.
median_val <- median(care_homes$staff_per_10)
mad_val <- mad(care_homes$staff_per_10, constant = 1)
care_homes$modified_z <- 0.6745 * (care_homes$staff_per_10 - median_val) / mad_val
care_homes$outlier_modz <- abs(care_homes$modified_z) > 3.5

# IQR
Q1 <- quantile(care_homes$staff_per_10, 0.25)
Q3 <- quantile(care_homes$staff_per_10, 0.75)
IQR_val <- IQR(care_homes$staff_per_10)
care_homes$outlier_iqr <- care_homes$staff_per_10 < (Q1 - 1.5*IQR_val) | 
                          care_homes$staff_per_10 > (Q3 + 1.5*IQR_val)

care_homes$num_flags <- care_homes$outlier_z + care_homes$outlier_modz + care_homes$outlier_iqr

from scipy import stats
from scipy.stats import median_abs_deviation

care_homes['z_score'] = stats.zscore(care_homes['staff_per_10'])
care_homes['outlier_z'] = abs(care_homes['z_score']) > 2.5

# Modified Z-score. NOTE: scipy's median_abs_deviation returns the RAW MAD
# by default, so the 0.6745 multiplier is required here (unlike R's
# pre-scaled mad()).
median_val = care_homes['staff_per_10'].median()
mad_val = median_abs_deviation(care_homes['staff_per_10'])
care_homes['modified_z'] = 0.6745 * (care_homes['staff_per_10'] - median_val) / mad_val
care_homes['outlier_modz'] = abs(care_homes['modified_z']) > 3.5

Q1 = care_homes['staff_per_10'].quantile(0.25)
Q3 = care_homes['staff_per_10'].quantile(0.75)
IQR_val = Q3 - Q1
care_homes['outlier_iqr'] = ((care_homes['staff_per_10'] < Q1 - 1.5*IQR_val) | 
                             (care_homes['staff_per_10'] > Q3 + 1.5*IQR_val))

care_homes['num_flags'] = (care_homes['outlier_z'].astype(int) + 
                           care_homes['outlier_modz'].astype(int) + 
                           care_homes['outlier_iqr'].astype(int))

print("DETECTION RESULTS:")

28.8 Step 4: Compare Methods - and Add the Domain Rule

Method agreement orders the queue, but it is not the whole queue. Notice what the statistical rules just did with the impossible values: a home reporting essentially zero qualified staff sits only about 3.4 robust standard deviations from the median - just inside the modified z-score’s 3.5 line - so the near-zero entries are flagged by one method (the fences) at most. A care home cannot operate with no qualified staff; that knowledge belongs in the screen as an explicit domain rule alongside the statistical ones, exactly as the detection chapter recommends.

  • R
  • Python
# Domain rule: below 1 qualified staff per 10 residents is implausible or
# an immediate concern, whatever the statistical flags say
care_homes$domain_flag <- care_homes$staff_per_10 < 1

# Investigation queue: 2+ statistical methods, or the domain rule
queue <- care_homes[care_homes$num_flags >= 2 | care_homes$domain_flag,
                    c("home_id", "staff_per_10", "z_score", "num_flags",
                      "domain_flag", "care_type", "beds", "cqc_rating")]
queue <- queue[order(-queue$num_flags, -abs(queue$z_score)), ]

28.9 Step 5: Investigation and Treatment

  • R
  • Python
# Base R classification of the investigation queue
investigation <- queue
investigation$finding <- ifelse(
  investigation$staff_per_10 < 0.2, "Data error: Decimal missing",
  ifelse(investigation$staff_per_10 > 9, "Data error: Wrong unit",
  ifelse(investigation$care_type == "Dementia" & investigation$staff_per_10 > 4.5, 
         "Genuine: Specialist unit",
  ifelse(investigation$staff_per_10 < 1.8 & investigation$cqc_rating == "Inadequate",
         "Concern: Known issues",
  ifelse(investigation$staff_per_10 > 5 & investigation$beds < 30, 
         "Small home variation", "Requires investigation")))))

investigation$action <- ifelse(
  grepl("Data error", investigation$finding), "Exclude",
  ifelse(grepl("Concern", investigation$finding), "Flag for inspection",
         "Keep with note"))

def classify_finding(row):
    if row['staff_per_10'] < 0.2:
        return "Data error: Decimal missing"
    elif row['staff_per_10'] > 9:
        return "Data error: Wrong unit"
    elif row['care_type'] == "Dementia" and row['staff_per_10'] > 4.5:
        return "Genuine: Specialist unit"
    elif row['staff_per_10'] < 1.8 and row['cqc_rating'] == "Inadequate":
        return "Concern: Known issues"
    elif row['staff_per_10'] > 5 and row['beds'] < 30:
        return "Small home variation"
    return "Requires investigation"

def classify_action(finding):
    if "Data error" in finding:
        return "Exclude"
    elif "Concern" in finding:
        return "Flag for inspection"
    return "Keep with note"

investigation = queue.copy()
investigation['finding'] = investigation.apply(classify_finding, axis=1)
investigation['action'] = investigation['finding'].apply(classify_action)

28.11 Key Takeaways

What We Learned

  1. Multiple methods, read by agreement: the three detectors agreed unanimously on the high tail, and the agreement ordered the queue
  2. Statistical rules have blind spots: the impossible near-zero entries slipped under the modified z-score’s threshold and needed an explicit domain rule – “a home cannot run on zero qualified staff” is knowledge no formula contains
  3. Investigation decides, detection only nominates: the same flagged value resolved to a unit error, a specialist unit, or a genuine concern only after the context was checked
  4. Context matters: specialist units and small homes legitimately differ
  5. Document everything: findings and justifications recorded for QA

Recommendations for Inspection Team

Immediate action required: - Homes flagged with genuine staffing concerns - Data quality issues to be corrected by providers

Monitor: - Small homes with higher variation (natural) - Specialist units with elevated staffing (expected)

QA Documentation

Complete investigation log created including: - Detection method results - Investigation findings for each flagged home - Treatment decisions with justification - Final clean dataset with exclusion flags

28.12 Practical Tips

  1. Always visualise first: Understand distribution before applying methods
  2. Use multiple methods: Look for consensus, not single-method results
  3. Prioritise investigation: Focus on homes flagged by 2+ methods
  4. Contact providers: Many “outliers” are simple data entry errors
  5. Consider context: Small providers and specialists naturally vary
  6. Document thoroughly: Investigation log essential for QA and defensibility

## Scenario

You are a CQC analyst investigating staffing levels at adult social care homes. Recent submissions show some extreme staff-to-resident ratios that warrant investigation. The analysis will support:

- Identifying homes requiring immediate inspection (data errors vs genuine understaffing)
- Understanding natural variation in staffing models
- Informing risk-based targeting

**Analytical Question**: Which care homes have staffing ratios extreme enough to warrant investigation, and which extreme values are data errors versus genuine operational issues?

**Context**: Care homes report monthly staffing data. Most homes have 2-4 qualified staff per 10 residents, but you've seen values ranging from 0 to 12. Some may be data entry errors (wrong decimal point, missing zeros), others may reflect genuine operational models (specialist dementia units) or problems (understaffing due to recruitment issues).

---

By working through this example, you will learn to:

### Data Characteristics

**The planted truth** (knowing the right answer is what makes a simulation instructive):
- Most homes (300): typical staffing around 3.2 staff per 10 residents (standard deviation 0.8)
- Specialist units (15): genuinely higher staffing around 5.5
- Struggling homes (10): genuinely low staffing around 1.5
- Planted problem values (12): impossible entries near zero, suspected wrong-unit entries above 9, and a few high-but-plausible returns
- Small homes get extra noise: higher natural variation

## Step 3: Apply Detection Methods

::: {.panel-tabset group="language"}
## R
```{r}
#| eval: true
# Z-score
care_homes$z_score <- scale(care_homes$staff_per_10)[,1]
care_homes$outlier_z <- abs(care_homes$z_score) > 2.5

# Modified Z-score. NOTE: R's mad() is already scaled to mimic the standard
# deviation (constant = 1.4826), so use the raw MAD (constant = 1) with the
# 0.6745 multiplier - applying both scalings under-flags by a third.
median_val <- median(care_homes$staff_per_10)
mad_val <- mad(care_homes$staff_per_10, constant = 1)
care_homes$modified_z <- 0.6745 * (care_homes$staff_per_10 - median_val) / mad_val
care_homes$outlier_modz <- abs(care_homes$modified_z) > 3.5

# IQR
Q1 <- quantile(care_homes$staff_per_10, 0.25)
Q3 <- quantile(care_homes$staff_per_10, 0.75)
IQR_val <- IQR(care_homes$staff_per_10)
care_homes$outlier_iqr <- care_homes$staff_per_10 < (Q1 - 1.5*IQR_val) | 
                          care_homes$staff_per_10 > (Q3 + 1.5*IQR_val)

care_homes$num_flags <- care_homes$outlier_z + care_homes$outlier_modz + care_homes$outlier_iqr

cat("DETECTION RESULTS:\n")
cat("Z-score:", sum(care_homes$outlier_z), "outliers\n")
cat("Modified Z:", sum(care_homes$outlier_modz), "outliers\n")
cat("IQR:", sum(care_homes$outlier_iqr), "outliers\n")
```

## Python
```{python}
#| eval: true
from scipy import stats
from scipy.stats import median_abs_deviation

care_homes['z_score'] = stats.zscore(care_homes['staff_per_10'])
care_homes['outlier_z'] = abs(care_homes['z_score']) > 2.5

# Modified Z-score. NOTE: scipy's median_abs_deviation returns the RAW MAD
# by default, so the 0.6745 multiplier is required here (unlike R's
# pre-scaled mad()).
median_val = care_homes['staff_per_10'].median()
mad_val = median_abs_deviation(care_homes['staff_per_10'])
care_homes['modified_z'] = 0.6745 * (care_homes['staff_per_10'] - median_val) / mad_val
care_homes['outlier_modz'] = abs(care_homes['modified_z']) > 3.5

Q1 = care_homes['staff_per_10'].quantile(0.25)
Q3 = care_homes['staff_per_10'].quantile(0.75)
IQR_val = Q3 - Q1
care_homes['outlier_iqr'] = ((care_homes['staff_per_10'] < Q1 - 1.5*IQR_val) | 
                             (care_homes['staff_per_10'] > Q3 + 1.5*IQR_val))

care_homes['num_flags'] = (care_homes['outlier_z'].astype(int) + 
                           care_homes['outlier_modz'].astype(int) + 
                           care_homes['outlier_iqr'].astype(int))

print("DETECTION RESULTS:")
print(f"Z-score: {care_homes['outlier_z'].sum()} outliers")
print(f"Modified Z: {care_homes['outlier_modz'].sum()} outliers")
print(f"IQR: {care_homes['outlier_iqr'].sum()} outliers")
```

## Step 4: Compare Methods - and Add the Domain Rule

Method agreement orders the queue, but it is not the whole queue. Notice what the statistical rules just did with the *impossible* values: a home reporting essentially zero qualified staff sits only about 3.4 robust standard deviations from the median - just inside the modified z-score's 3.5 line - so the near-zero entries are flagged by one method (the fences) at most. A care home cannot operate with no qualified staff; that knowledge belongs in the screen as an explicit **domain rule** alongside the statistical ones, exactly as [the detection chapter](../chapters/16-outlier-methods.qmd) recommends.

::: {.panel-tabset group="language"}
## R
```{r}
#| eval: true
# Domain rule: below 1 qualified staff per 10 residents is implausible or
# an immediate concern, whatever the statistical flags say
care_homes$domain_flag <- care_homes$staff_per_10 < 1

# Investigation queue: 2+ statistical methods, or the domain rule
queue <- care_homes[care_homes$num_flags >= 2 | care_homes$domain_flag,
                    c("home_id", "staff_per_10", "z_score", "num_flags",
                      "domain_flag", "care_type", "beds", "cqc_rating")]
queue <- queue[order(-queue$num_flags, -abs(queue$z_score)), ]

cat("INVESTIGATION QUEUE (2+ methods or domain rule):", nrow(queue), "homes\n\n")
print(queue, row.names = FALSE)
```

## Python
```{python}

care_homes['domain_flag'] = care_homes['staff_per_10'] < 1

queue = care_homes[(care_homes['num_flags'] >= 2) | care_homes['domain_flag']][
    ['home_id', 'staff_per_10', 'z_score', 'num_flags', 'domain_flag',
     'care_type', 'beds', 'cqc_rating']
].sort_values(['num_flags', 'z_score'], key=abs, ascending=False)

print(f"INVESTIGATION QUEUE (2+ methods or domain rule): {len(queue)} homes\n")
print(queue.to_string(index=False))
```

## Step 5: Investigation and Treatment

::: {.panel-tabset group="language"}
## R
```{r}
#| eval: true
# Base R classification of the investigation queue
investigation <- queue
investigation$finding <- ifelse(
  investigation$staff_per_10 < 0.2, "Data error: Decimal missing",
  ifelse(investigation$staff_per_10 > 9, "Data error: Wrong unit",
  ifelse(investigation$care_type == "Dementia" & investigation$staff_per_10 > 4.5, 
         "Genuine: Specialist unit",
  ifelse(investigation$staff_per_10 < 1.8 & investigation$cqc_rating == "Inadequate",
         "Concern: Known issues",
  ifelse(investigation$staff_per_10 > 5 & investigation$beds < 30, 
         "Small home variation", "Requires investigation")))))

investigation$action <- ifelse(
  grepl("Data error", investigation$finding), "Exclude",
  ifelse(grepl("Concern", investigation$finding), "Flag for inspection",
         "Keep with note"))

cat("INVESTIGATION SUMMARY:\n\n")
table(investigation$action)
```

## Python
```{python}
#| eval: true
def classify_finding(row):
    if row['staff_per_10'] < 0.2:
        return "Data error: Decimal missing"
    elif row['staff_per_10'] > 9:
        return "Data error: Wrong unit"
    elif row['care_type'] == "Dementia" and row['staff_per_10'] > 4.5:
        return "Genuine: Specialist unit"
    elif row['staff_per_10'] < 1.8 and row['cqc_rating'] == "Inadequate":
        return "Concern: Known issues"
    elif row['staff_per_10'] > 5 and row['beds'] < 30:
        return "Small home variation"
    return "Requires investigation"

def classify_action(finding):
    if "Data error" in finding:
        return "Exclude"
    elif "Concern" in finding:
        return "Flag for inspection"
    return "Keep with note"

investigation = queue.copy()
investigation['finding'] = investigation.apply(classify_finding, axis=1)
investigation['action'] = investigation['finding'].apply(classify_action)

print("INVESTIGATION SUMMARY:\n")
print(investigation['action'].value_counts())
```

## Key Takeaways

### What We Learned

1. **Multiple methods, read by agreement**: the three detectors agreed unanimously on the high tail, and the agreement ordered the queue
2. **Statistical rules have blind spots**: the impossible near-zero entries slipped under the modified z-score's threshold and needed an explicit domain rule – "a home cannot run on zero qualified staff" is knowledge no formula contains
3. **Investigation decides, detection only nominates**: the same flagged value resolved to a unit error, a specialist unit, or a genuine concern only after the context was checked
4. **Context matters**: specialist units and small homes legitimately differ
5. **Document everything**: findings and justifications recorded for QA

### Recommendations for Inspection Team

**Immediate action required**:
- Homes flagged with genuine staffing concerns
- Data quality issues to be corrected by providers

**Monitor**:
- Small homes with higher variation (natural)
- Specialist units with elevated staffing (expected)

### QA Documentation

Complete investigation log created including:
- Detection method results
- Investigation findings for each flagged home
- Treatment decisions with justification
- Final clean dataset with exclusion flags

---

## Practical Tips

1. **Always visualise first**: Understand distribution before applying methods
2. **Use multiple methods**: Look for consensus, not single-method results
3. **Prioritise investigation**: Focus on homes flagged by 2+ methods
4. **Contact providers**: Many "outliers" are simple data entry errors
5. **Consider context**: Small providers and specialists naturally vary
6. **Document thoroughly**: Investigation log essential for QA and defensibility
